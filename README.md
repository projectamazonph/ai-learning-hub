# AI Ready PH Learning Hub

### Learn AI by doing real work. Get evidence-based feedback. Build proof you can actually use it.

---

## The Problem

Most AI courses teach you what buttons to click. You finish a course, watch a few tutorials, maybe copy some prompt templates — and then you still can&rsquo;t confidently handle a client brief, write a proper business case, or debug your own AI-assisted workflow.

The uncomfortable truth: **watching someone else use AI is not the same as learning to use it yourself.**

The other problem: Filipino VAs, freelancers, and business owners are being told to &ldquo;learn AI&rdquo; but nobody tells them *which* AI skills actually matter for their work, or how to prove competence without a portfolio.

---

## The Solution

**AI Ready PH Learning Hub** is not a video library with a chatbot in the corner.

It is a practice-first learning environment built around one promise:

> **Turn AI confusion into practical skills, repeatable workflows, and proof you can do the work.**

You learn by completing real tasks, receiving rubric-based feedback, and building a portfolio of evidence — not just a certificate that says you clicked &ldquo;Complete&rdquo; on 40 slides.

---

## What Makes This Different

### 🎯 Outcomes Before Tools

We don&rsquo;t teach &ldquo;ChatGPT tips and tricks.&rdquo; We teach transferable thinking:

- **Task framing** — converting a vague client request into a clear brief
- **Prompt design as work design** — instructions, context, examples, and quality checks
- **Research and verification** — separating facts from confident hallucinations
- **Output quality assurance** — reviewing correctness, completeness, tone, and risk
- **Workflow design** — knowing what to automate, what to assist, and what to keep human

Tool brands come and go. These skills transfer.

### 🧪 Practice Before Confidence

Every important concept ends in a task that produces real evidence:

- Write a brief → get rubric feedback
- Build a workflow → stress-test it with injected failures
- Analyze data → submit a deliverable with reasoning notes
- Run a research session → cite sources, catch contradictions, flag unknowns

If you can&rsquo;t do the work, you don&rsquo;t get the badge. Simple as that.

### 📋 Evidence-Based Progression

Completion percentage is not a metric here.

We track **11 competencies across 5 mastery levels:**

| Level | What It Means |
|---|---|
| **Aware** | Recognizes the concept |
| **Guided** | Completes the task with step-by-step support |
| **Independent** | Completes a normal task without hints |
| **Adaptive** | Handles ambiguity and adjusts the method |
| **Operational** | Creates a reusable process and can teach or govern it |

A certificate requires demonstrated competence — not just clicking through every page.

### 📱 Mobile-First, Philippine-Ready

Core lessons and most labs work at 390px. Desktop-only tasks are clearly labeled. Philippine payment methods (GCash, Maya, bank transfer) are supported natively. No PayPal required.

## The Lab System

This is where the learning actually happens. **Nine practice environments** — not quizzes, not videos — hands-on labs where you do the work:

| Lab | What You Do |
|---|---|
| **Prompt Workbench** | Build prompts from objective, audience, inputs, and constraints. Compare versions side by side. Score instruction fit and output stability. |
| **Brief-to-Deliverable Studio** | Turn messy client or business requests into clear briefs. Generate emails, reports, plans, and content. Pass QA before submission. |
| **Research &amp; Verification Lab** | Plan research before running it. Review source packs. Catch invented citations and unresolved contradictions. |
| **Output QA Lab** | Diagnose flawed AI output. Mark factual, logical, formatting, and privacy defects. Revise and explain each correction. |
| **Workflow Canvas** | Place human, AI, and system steps on a visual workflow. Define inputs, owners, approvals, and failure states. |
| **Automation &amp; Agent Simulator** | Assemble triggers, decisions, tools, and approval checkpoints. Run synthetic scenarios with injected failures. No real side effects. |
| **Data Safety Clinic** | Classify sample data. Decide what can enter an AI tool. Detect prompt injection and select safe alternatives. |
| **ROI &amp; Opportunity Mapper** | Inventory recurring work. Score opportunities. Produce a prioritized adoption backlog. |
| **Portfolio Builder** | Convert selected lab work into polished case studies. Generate a public share page with privacy controls. |

**Three execution modes for every lab:**

| Mode | Purpose | Cost |
|---|---|---|
| **Demo** | Show a complete example with precomputed responses | Free |
| **Practice** | Experiment and receive guidance | Metered credits |
| **Assessment** | Evaluate under controlled rules | Version-pinned, stricter limits |

## The 11 Competencies

Every learner is measured on these transferable skills — not completion percentage:

| Competency | What You Must Demonstrate |
|---|---|
| **AI literacy** | Explain strengths, limitations, uncertainty, and suitable uses |
| **Task framing** | Convert a vague request into a clear objective, inputs, constraints, and acceptance criteria |
| **Prompt design** | Give useful instructions, context, examples, formats, and evaluation checks |
| **Context management** | Select, prepare, and safely provide the right source material |
| **Research &amp; verification** | Find evidence, check sources, identify unsupported claims, resolve conflicts |
| **Output QA** | Review correctness, completeness, tone, format, and risk before delivery |
| **Workflow design** | Break work into steps with clear human and AI responsibilities |
| **Automation judgment** | Decide what should be automated, assisted, or kept human |
| **Data safety &amp; ethics** | Handle private, copyrighted, sensitive, and biased material appropriately |
| **Client communication** | Explain decisions, limitations, changes, and required approvals clearly |
| **Business value** | Connect an AI workflow to time, cost, revenue, quality, or risk improvement |
