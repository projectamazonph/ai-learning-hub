# Review the Current Change

Review the current change without implementing fixes.

1. Read `AGENTS.md`, the issue, source documents, diff, and tests.
2. Identify the intended user outcome.
3. Map every acceptance criterion to code and test evidence.
4. Inspect for correctness, state transitions, failure recovery, authorization, tenant scope, idempotency, concurrency, privacy, security, accessibility, mobile layout, performance, content freshness, and AI cost.
5. Run relevant checks when available.
6. Report findings ordered by severity.

Each finding must include exact location, behavior, impact, reproduction or reasoning, and recommended correction. Separate confirmed defect, material risk, and optional improvement.

End with a go, conditional go, or no-go recommendation and the commands actually run.
