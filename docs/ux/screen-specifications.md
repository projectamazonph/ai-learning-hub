# Screen Specifications

## Shared page states

Every data-dependent screen defines:

- Loading.
- Empty.
- Ready.
- Partial or degraded.
- Validation error.
- Authorization denied.
- Not found.
- Retryable failure.
- Final failure.

Skeletons reflect the real layout. Empty states explain the next valid action. Errors do not erase entered work.

## Landing page

### Purpose

Make the applied learning difference clear and move the right visitor to a first useful result.

### Sections

1. Hero: promise, audience, primary and secondary calls to action.
2. Learning standard: frame, generate, verify, improve.
3. Role paths.
4. Practice-lab preview.
5. Skills Passport and proof.
6. How the first win works.
7. Pricing preview.
8. FAQ.
9. Final call to action.

### Mobile

- Hero actions stack full width.
- Manual card remains legible without decorative overflow.
- One call to action per viewport.
- No auto-playing video.

## Readiness diagnostic

### Anatomy

- Progress and estimated remaining time.
- One question per step on mobile.
- Plain-language reason for sensitive questions.
- Back, save, and continue.
- Final recommendation with reasons and override.

### Rules

- Eight questions maximum.
- Optional answers labeled.
- Recommendation does not imply employment outcome.
- A visitor can see a result before payment.

## Learner dashboard

### Priority order

1. Recommended next action.
2. Why it matters and expected time.
3. Current path and progress.
4. Competency movement.
5. Recent evidence and feedback.
6. Credits and account notices.

### Empty state

New learners see the diagnostic or first-win action, not zeros and charts.

## Learning path map

### Anatomy

- Path outcome and capstone.
- Module sequence with prerequisites.
- Current module.
- Completed, available, locked, and optional states.
- Competency coverage.
- Estimated remaining effort.

### Mobile

Use a vertical ordered map. Do not shrink a desktop node graph into unreadable confetti.

## Lesson reader

### Anatomy

- Breadcrumb and progress.
- Outcome, time, competency, reviewed date.
- Content blocks.
- Sticky table of contents on desktop.
- Bottom-sheet outline on mobile.
- Required interaction.
- Previous and next actions.

### Rules

- Reading width stays near 70 characters.
- Video has captions and transcript.
- Source references are available without breaking reading flow.
- Completion state confirms which action satisfied the lesson.

## Lab catalog

### Filters

- Skill.
- Role.
- Difficulty.
- Estimated time.
- Demo, practice, or assessment.
- Available entitlement.

### Card

- Lab name and number.
- Skill outcome.
- Scenario count.
- Mode and cost behavior.
- Accessibility or desktop requirement.
- Resume state when a run exists.

## Lab runner

### Desktop layout

- Scenario brief and sources.
- Active task workspace.
- Checks, rubric, and run status.
- Save and submit controls.

### Mobile layout

- One active region at a time.
- Tabs or step navigation for brief, work, and review.
- Sticky save or submit action.
- Confirmed-save indicator.
- No side-by-side comparison that forces horizontal scroll.

### Required controls

- Save.
- Resume.
- Request review.
- Submit.
- Cancel live run.
- Retry when safe.
- Start revision.
- View version history.
- Export learner-owned final artifact.

### Live AI strip

Always show:

- Demo or live mode.
- Model label appropriate for the learner.
- Scenario version.
- Estimated or actual credit use.
- Data boundary.
- Run status.

## Evaluation view

### Anatomy

- Overall status and score.
- Criterion scores.
- Evidence for each score.
- Priority feedback.
- Safety or integrity flags.
- First versus final comparison.
- Next action.

Scores never appear without explanation.

## Portfolio builder

### Sections

- Problem.
- Context and constraints.
- Process.
- Human and AI responsibilities.
- Evidence and verification.
- Result.
- Reflection.
- Privacy and synthetic-data label.

### Preview

The learner sees exactly what public viewers will see. Raw private files are excluded by default.

## Skills Passport

### Anatomy

- Competency level and evidence count.
- Selected evidence cards.
- Course and certificate links.
- Verification status.
- Privacy setting.

Avoid radar charts as the only representation. Provide exact levels and evidence in text.

## Pricing and checkout

### Pricing card

- Price and access period.
- Included paths and labs.
- Live credit allowance.
- Assessment and review limits.
- Refund summary.
- Team seat behavior where relevant.

### Checkout

- Product and price.
- Discount.
- Payment method.
- Terms, refund, and AI-credit notice.
- Billing identity.
- Final explicit payment action.

Success depends on verified payment state, not browser return URL alone.

## Workspace overview

- Seat and invitation status.
- Assignment activity.
- Aggregate competency movement.
- Allowed learner progress.
- Credit usage and limits.
- Policy acknowledgement.

Manager views do not expose private practice content by default.

## Admin review queue

- Submission age and service target.
- Learner and path.
- Scenario and rubric version.
- Automated checks and policy flags.
- Reviewer assignment.
- Approve, note, revise, reject-policy, and escalate actions.

Every decision requires criterion feedback or a reason code.

## Admin content freshness

- Overdue count by volatility.
- Content ranked by risk, traffic, and assessment use.
- Sources and last decision.
- Start review, revise, publish, supersede, or retire.
- Model change and calibration impact.
