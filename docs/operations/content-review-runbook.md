# Content Review Runbook

## Daily or scheduled review

1. Open the content freshness queue.
2. Sort by overdue severity, volatility, assessment use, learner traffic, and policy sensitivity.
3. Assign owner and due target.
4. Do not bulk-mark reviewed without source inspection.

## Review one item

1. Read the lesson outcome and supported competencies.
2. Inspect current source list and prior review decision.
3. Verify each volatile factual claim against an authoritative source.
4. Check screenshots, steps, plan limits, model names, and regional behavior.
5. Check examples, quiz explanations, linked labs, and templates.
6. Decide:
   - No material change.
   - Revision required.
   - Safety correction required.
   - Assessment calibration required.
   - Retire and replace.
7. Record sources, evidence, decision, and next review date.

## Publish revision

- Create a new immutable content version.
- Update source references and reviewed date.
- Keep evergreen method separate from volatile walkthrough.
- Run content validation.
- Run assessment calibration when behavior changes.
- Check mobile, accessibility, and localization impact.
- Publish and link superseded version.

## Urgent correction

For unsafe or materially false guidance:

- Unpublish or add a blocking notice.
- Identify affected learners and assessments.
- Correct and review.
- Reissue or invalidate evidence only through an explicit governed decision.
- Communicate clearly when learner action is required.

## Weekly report

- Due and overdue count by volatility.
- High-risk overdue items.
- Published revisions.
- Retired items.
- Assessment calibrations.
- Source or provider changes requiring wider action.
