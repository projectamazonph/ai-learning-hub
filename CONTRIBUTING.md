# Contributing

1. Read `AGENTS.md`, `AI_CONTEXT.md`, `WORKSPACE.md`, and the relevant source document.
2. Create one issue or story for one user outcome.
3. Add acceptance criteria and failure cases.
4. Write or update tests first.
5. Make the smallest passing implementation.
6. Run proportional checks and inspect changed UI.
7. Update project records.
8. Submit one focused pull request.

## Commit format

Use conventional commits with a story reference when one exists:

```text
feat(labs): add scenario version guard (STORY-024)
fix(credits): reconcile cancelled reservations (STORY-031)
docs(curriculum): define source verification rubric
```

No agent may commit or publish changes without explicit user authorization.
